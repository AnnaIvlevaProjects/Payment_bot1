# Telegram payment bot

Бот на `aiogram` + `SQLite` для помесячной оплаты курса (6 месяцев).

## Что умеет
- `/start`:
  - приветствует пользователя;
  - сохраняет в SQLite: `user_id`, `user_name`, `user_FN`, `user_LN`, `source` (из deep-link аргумента).
- Кнопка **«О курсе»**: отправляет структуру курса.
- Кнопка **«Оплатить»**:
  - показывает реквизиты;
  - предлагает inline-кнопки выбора: `1..6 месяц` и **«Весь курс»**;
  - после выбора просит загрузить чек.
- При загрузке чека:
  - принимает документ/изображение (`.txt`, `.pdf`, `.png`, `.jpg`, `.jpeg` и др. популярные форматы изображений);
  - отправляет чек админу с данными пользователя;
  - записывает оплату в `payment_N = "да"`;
  - если выбран **«Весь курс»**, записывает `да` во все `payment_1..payment_6`;
  - отправляет пользователю ссылку на учебный чат;
  - предлагает оставить e-mail (с валидацией).
- Если пользователь загрузил файл слишком рано (до выбора месяца/всего курса), бот подсказывает пользоваться меню.

## Логика проверок оплат (по вашему сценарию)
- Дата старта курса задаётся через `COURSE_START_DATE` (по умолчанию `2026-04-04`).
- Проверки идут для 5 периодов (май–сентябрь 2026 при старте 04.04.2026):
  - за **3 дня до даты удаления**: если нет оплаты за следующий месяц (`payment_2..payment_6`), отправляется напоминание;
  - в **дату удаления периода**: если этой оплаты нет, пользователь удаляется из учебного чата и получает уведомление.
- Текущий период определяется по разнице дат от `COURSE_START_DATE`: новый период начинается в сам день-границу (например, при старте `04.MM` второй период начинается `04.(MM+1)`, а не `05`).
- Если после удаления пользователь загружает чек — доступ в чат возвращается автоматически.

## Редактирование текстов без программиста
Все крупные тексты вынесены в `messages.json`:
- приветствие;
- блок «О курсе»;
- реквизиты;
- уведомления/напоминания.

Менять формулировки можно в `messages.json` без изменений в Python-коде.

## Установка
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

Заполните `.env`:
- `BOT_TOKEN` — токен бота;
- `ADMIN_CHAT_ID` — ID админа/группы для чеков;
- `COURSE_CHAT_ID` — ID закрытого чата/канала курса;
- `COURSE_CHAT_LINK` — ссылка-приглашение в чат;
- `DB_PATH` — путь до SQLite;
- `CHECK_INTERVAL_HOURS` — интервал фоновой проверки;
- `DB_EXPORT_INTERVAL_HOURS` — как часто проверять, не пора ли делать суточную выгрузку БД;
- `DB_EXPORT_PATH` — папка, куда сохраняются XLSX-выгрузки пользователей;
- `YANDEX_DISK_TOKEN` — OAuth-токен Яндекс.Диска для автоматической загрузки выгрузок;
- `YANDEX_DISK_EXPORT_PATH` — путь на Яндекс.Диске для выгрузок (по умолчанию `app:/payment_bot_exports`);
- `COURSE_START_DATE` — дата старта обучения в формате `YYYY-MM-DD`;
- `MESSAGES_FILE` — путь до файла с текстами (по умолчанию `messages.json`).

## Ежедневная выгрузка БД
- Бот раз в сутки формирует XLSX-таблицу `users_export_YYYY-MM-DD.xlsx` со всеми полями пользователей.
- Файл сохраняется в папку `DB_EXPORT_PATH` (по умолчанию `exports`).
- После формирования бот отправляет XLSX в `ADMIN_CHAT_ID`.
- При наличии `YANDEX_DISK_TOKEN` бот автоматически загружает тот же XLSX на Яндекс.Диск в `YANDEX_DISK_EXPORT_PATH` (вручную переносить в онлайн-хранилище не нужно).
- Частота фоновой проверки на новый день задается `DB_EXPORT_INTERVAL_HOURS` (по умолчанию `1`).

## Запуск
```bash
python bot.py
```


## Как скопировать полный файл (а не patch)
Если нужен именно целый файл из репозитория, используйте одну из команд:

```bash
# Показать полный текущий файл в терминале
cat bot.py

# Скопировать файл как есть в другое место
cp bot.py /tmp/bot.py

# Вытащить полный файл из конкретного коммита
git show <commit_sha>:bot.py > bot_from_commit.py

# Сохранить текущую версию файла из HEAD
git show HEAD:bot.py > bot_head.py
```

`git diff` и patch показывают только изменения, а команды выше — полный контент файла.


## Как протестировать удаление/возврат при неуплате
1. Поставьте короткий интервал, например `CHECK_INTERVAL_HOURS=1`.
2. В тестовой БД вручную выставьте для пользователя `payment_1='нет'`.
3. В день проверки (1-е или 4-е число) запустите бота и дождитесь цикла.
4. Убедитесь, что пришло напоминание (1-го) или произошло удаление (4-го).
5. Затем загрузите чек через бота и проверьте, что:
   - в БД нужная оплата стала `да`;
   - бот вернул пользователя в учебный чат.

Для ускоренного локального теста можно временно поменять `COURSE_START_DATE` на дату, при которой текущий месяц попадает в окно проверок.

## Структура БД
Таблица `users` хранит:
- обязательные поля по ТЗ: `user_id`, `user_name`, `user_FN`, `user_LN`, `user_email`, `source`, `payment_1..payment_6`;
- служебные поля для логики доступа: `course_start_date`, `selected_month`, `removed_from_chat`, `last_reminder_month`, `last_removal_month`, timestamps.

## Важно
- Бот должен быть админом в учебном чате, чтобы удалять/возвращать участников.
- Для `source` используйте deep-link вида:
  `https://t.me/<bot_username>?start=utm_source_google_campaign1`
